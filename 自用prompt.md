# 需求说明：AnimationPlatform 动画演示层视觉升级方案

我是当前的开发负责人。目前我们正在开发基于 Unity (2022.3.53f1c1) 的 `AnimationPlatform` 层级。这个层级主要用于宏观的动画演示，场景内包含 **3 个机械臂** 和 **3 只仿真鸟**。机械臂末端的 `TrailRenderer` 拖尾效果已经完成。

请你结合我们本地的代码结构和 URP 渲染管线，帮我评估并实现以下三个视觉与前端升级方案。请在阅读完毕后，先给我一个落地的执行计划，确认无误后再开始写代码或修改预制体。

## 升级目标一：基于 Cinemachine 的运镜调度
由于场景中有 3 组设备，固定的摄像机或纯手动的操作无法满足“自动化展示”的需求。
- **任务：** 引入 Cinemachine (已升级至 2.10.7)，在场景中建立 `Dolly Track` (轨道)。
- **逻辑：** 1. 默认状态下，相机沿轨道缓慢环绕 3 个机械臂和鸟，作为“自动演示模式”。
  2. 需要配置一个 `Target Group` 包含这 3 只仿真鸟，确保相机在环绕时能自动根据鸟的飞散程度推拉镜头（调整 FOV 或轨道距离）。
  3. 当用户通过鼠标拖拽交互时，平滑退出轨道模式，切换为自由视角（我们已有的拖拽功能）；鼠标停止操作一段时间后，自动恢复轨道环绕。

## 升级目标二：3D 空间连线 (阵型可视化)
为了增加数字孪生的科技感，我们需要用线将 3 组设备连接起来。
- **任务：** 使用 `LineRenderer` 在 3 只仿真鸟（或机械臂末端）之间绘制闭合的三角形阵型连线。
- **要求：**
  - 材质使用 URP 下带有 Emission 的发光材质（如科技蓝），支持半透明。
  - 需要一个专门的 C# 脚本（例如 `FormationVisualizer.cs`）在 `Update` 中实时更新这三个顶点的位置。

## 升级目标三：World Space UI 数据悬浮窗
在刚才建立的阵型中心，悬浮一个展示数据的 UI 面板。
- **任务：** 建立一个 `World Space` 模式的 Canvas（不使用 UI Toolkit，因为这是空间内跟随的 UI）。
- **要求：**
  - Canvas 包含一个半透明背景和 TextMeshPro 文本（用于显示“协同误差: 0.02mm”等模拟数据）。
  - 该 UI 必须通过脚本实时计算自身位置（位于 3 只鸟坐标的中心点，并向上偏移一点）。
  - 该 UI 必须具备 **Billboard (广告牌)** 效果，即在 `Update` 中始终朝向主摄像机（`Camera.main`），确保不管 Cinemachine 怎么绕，文字始终正对屏幕。
